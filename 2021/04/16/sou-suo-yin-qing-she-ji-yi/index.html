数据爬取

数据来源

  • 在构建电影搜索引擎之前，必须收集电影相关数据。www.imdb.com是一个电影网站，其中包含按电影名称，导演，主演以及电影介绍等相关信息。

  • 利用Jupyter Notebook编写一个web crawler来爬取2017-2021年共9990条数据。

    Note：别问我为啥是9990条，本来爬取了1w条，结果最后少了10条 (T_T)

  • 由于电影的数据种类很多，我们就只爬取了电影名，类型，评分，导演，主演以及评论。对于原数据为空的，我们添加None。

import re
import json
import requests
from bs4 import BeautifulSoup

from google.colab import drive
drive.mount('/content/drive/', force_remount=True)

movie_count = 1
page_count = 1
Max_page_number = 40
movie_json = []
while page_count <= Max_page_number:
  print('Page:',page_count)
  if page_count == 1:
    url = 'https://www.imdb.com/search/title/?title_type=tv_movie&release_date=2017-01-01,2021-01-01&sort=user_rating,desc&count=250'
  else:
    url = 'https://www.imdb.com/search/title/?title_type=tv_movie&release_date=2017-01-01,2021-01-01&sort=user_rating,desc&count=250&start='+str((page_count-1)*250+1)+'&ref_=adv_nxt'
    # Read the content of the original json file to prevent the connection from being interrupted and data loss
    print('**********reading json file**********')
    movie_json = []
    with open('/content/drive/My Drive/movie_data.json', 'r') as f:
        load_dict = json.load(f)
        num_movie = len(load_dict)
        print(num_movie,'movies in document')
        print('start Crawler from',(page_count-1)*250+1,'movie later')
        for i in range(num_movie):
            movie_name = load_dict[i]['movie_name']
            movie_type = load_dict[i]['movie_type']
            movie_score = load_dict[i]['movie_score']
            director = load_dict[i]['director']
            stars = load_dict[i]['stars']
            movie_synopsis = load_dict[i]['movie_synopsis']
            movie_dict = &#123;'movie_name': movie_name,'movie_type': movie_type,'movie_score': movie_score,'director': director,'stars': stars,'movie_synopsis': movie_synopsis&#125;
            movie_json.append(movie_dict)
    print('***************finish****************')
  page_count += 1

  html = requests.get(url)
  html.raise_for_status()
  print(html)
  soup = BeautifulSoup(html.content, 'lxml')
  movies = soup.findAll(class_="lister-item mode-advanced")
  for movie in movies:
    movie_name = 'None'
    movie_type = 'None'
    movie_score = 'None'
    director = 'None'
    stars = 'None'
    movie_synopsis = 'None'


    # movie_name
    if movie.select_one('.lister-item-header').a is None:
      movie_name = 'None'
    else:
      movie_name = movie.select_one('.lister-item-header').a.string

    # movie_type
    if movie.select_one('.genre') is None:
      movie_type = 'None'
    else:
      clean_type = movie.select_one('.genre').string.replace('\n', '')
      clean_type = clean_type.strip()
      movie_type = clean_type

    # movie_score
    if movie.select_one('.ratings-bar') is None:
      movie_score = 'None'
    elif movie.select_one('.ratings-bar').strong is None:
      movie_score = 'None'
    else:
      movie_score = movie.select_one('.ratings-bar').strong.string

    # director and stars
    movie_url = 'https://www.imdb.com'+movie.select_one('.lister-item-header').a.get("href")+'?ref_=adv_li_tt'
    movie_page = requests.get(movie_url)
    movie_soup = BeautifulSoup(movie_page.content, 'lxml')
    for staff in movie_soup.findAll(class_="credit_summary_item"):
      if staff.h4.string == "Director:" or staff.h4.string == "Directors:":
        director = ''
        key = 0
        for director_name in staff.findAll('a'):
          if key < 3:
            director = director + director_name.string + ','
          key += 1
      if staff.h4.string == "Star:" or staff.h4.string == "Stars:":
        stars = ''
        key = 0
        for stars_name in staff.findAll('a'):
          if key < 3:
            stars = stars + stars_name.string + ','
          key += 1

    # movie_synopsis
    if movie.select_one('.lister-item-header').a.get("href") is None:
      movie_synopsis = 'None'
    else:
      movie_sum = 'https://www.imdb.com'+movie.select_one('.lister-item-header').a.get("href")+'plotsummary?ref_=tt_ov_pl'
      movie_sum_page = requests.get(movie_sum)
      movie_sum_soup = BeautifulSoup(movie_sum_page.content, 'lxml')
      movie_synopsis = movie_sum_soup.select_one('.ipl-zebra-list__item').p.string


    #Generate per-movie information
    movie_content = &#123;
        'movie_name': movie_name,
        'movie_type': movie_type,
        'movie_score': movie_score,
        'director': director,
        'stars': stars,
        'movie_synopsis': movie_synopsis
    &#125;

    if movie_count % 50 == 0:
      print('No.',movie_count,'Movie')
    movie_count += 1
    movie_json.append(movie_content)

  # insert to json file
  with open('/content/drive/My Drive/movie_data.json', 'w', encoding='utf8') as json_file:
    json.dump(movie_json,json_file,ensure_ascii=False)

配置ElasticSearch

  • 下载并安装Elasticsearch，打开安装后Elasticsearch的bin文件夹，然后双击运行elasticsearch.bat。

    这步可以在本地主机中托管elasticsearch，默认端口是9200。

D:\elasticsearch-7.11.1\bin\elasticsearch.bat
  • 打开Web浏览器，然后粘贴以下URL，以检查Elasticsearch搜索服务器是否已启动并正在运行。
http://localhost:9200/
  • 结果必须类似于下图，说明运行成功。

  • 请注意，默认端口是9200，但可以更改。在config文件夹中，打开“ config.yml”文件，并根据需要编辑端口号。
http.port：9300

数据导入

  • 手动添加JSON将是一项繁重的任务，而且成本很高。因此，Elasticsearch借助“ BULK API”提供了帮助，我们可以在一个API调用中添加大量的documents。
  • 为了通过BulkAPI添加文档，需要将我通过抓取网站收集的JSON转换为Bulk JSON。但是需要添加一个小的Python脚本来对我收集的数据进行修改，并使用Python通过BulkAPI进行数据添加。

您可以自己尝试一下，只需运行下面的InsertDataToES.py。另外，如果您在读取文件和添加任何字符替换时具有自己的数据和操作，请尝试修改以下脚本。Index的创建，documents的添加和搜索也可以使用python和elasticsearch库完成。

import elasticsearch
from elasticsearch import Elasticsearch, helpers
from elasticsearch_dsl import Index
import json, re
import codecs
import unicodedata

# import queries
es = Elasticsearch(HOST="http://localhost", PORT=9200)

def read_all_movies():
    with open('movie_data.json', 'r', encoding='utf-8-sig') as f:
        all_movies = json.loads(f.read().replace("&#125;\n&#123;", "&#125;,\n&#123;"))
        res_list = [i for n, i in enumerate(all_movies) if i not in all_movies[n + 1:]]
        return res_list

def genData(movie_array):
    for movie in movie_array:

        # Fields-capturing                
        name = movie.get("movie_name", None)
        types = movie.get("movie_type",None)
        score = movie.get("movie_score", None)
        director = movie.get("director", None)
        stars = movie.get("stars", None)
        synopsis = movie.get("movie_synopsis", None)

        yield &#123;
            #"_index": "imdb_dataset",
            "_source": &#123;
                "movie_name": name,
                "movie_type": types,
                "movie_score":score,
                "director": director,
                "stars": stars,
                "movie_synopsis": synopsis,
            &#125;,
        &#125;

# createIndex()
all_movies = read_all_movies()
helpers.bulk(es,genData(all_movies))
  • 通过运行脚本将documents添加到BulkAPI。修改“index”名称以适合您的名称并运行python脚本。将数据添加到相应目录下并更新read_all_movies()method中的路径。如果要添加更多文档，请将它们添加到同一文件夹中并运行脚本。对于抓取的数据，我已经相应地定义了字段。但是，如果您对数据进行了任何更改，请在genData()方法中添加相同的字段。
  • 运行该ipynb文件，若输出结果如下所示，则运行成功：
(9990, [])

配置Kibana

为了进一步确认是否成功将json文件里的数据导入Elasticsearch里，我们利用Kibana来可视化dataset

  • 下载并安装Kibana，双击运行kibana.bat
D:\kibana-7.11.1-windows-x86_64\bin\kibana.bat
  • 打开Web浏览器，然后粘贴以下URL，打开Kibana
http://localhost:5601/
  • 从左侧的菜单栏选择Management -> Stack Management -> Data -> Index Management，若如下图所示，imdb_dataset即我们传入到Elasticsearch中的数据，共9990条，则成功！

  • 这个界面可以手动管理我们的dataset，包括后续我们为BM25F模型创建的新的dataset，都会在这里显示。
